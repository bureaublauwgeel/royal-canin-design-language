Forms

Usage and implementation for Forms within Web Design Language.

Introduction

Form elements will ensure rapid form development is a possibility.

Strict markup will ensure all form elements are displayed correctly

Examples

Please ensure that the for="" attribute is used on labels and that it matches the input ID, for accessibility purposes, for example: 

@@include('./01-components/forms/_input-standard.html', {
					"type": "text" 
				})

Since the for attribute relies on an ID to match up with, ensure that all Input IDs are unique on your page.

Every input, where applicable, must include a Label. If the design calls for an instance where no label exists, it should be hidden using the screen-reader-text class. More information on this can be on the Accessibility page.

When building a form, please use a wrapping element to position your Inputs, as opposed to applying positioning rules directly to the Inputs. For example:

@@include('./01-components/forms/_form-positioning.html')

Please don't use the Image input type for submit buttons.

The Submit input type, a button or an anchor with a class of button is more appropriate.
