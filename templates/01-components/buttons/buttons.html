Buttons

Usage and implementation for Buttons within Web Design Language.

Introduction

Buttons can be used for a variety of actions.

Sizing modifier classes can be used to change the size of the Button.

Button states can give different visual representations to the user.

Icons can be used within Buttons to hightlight a specific area or action.

Examples

A primary call to action should be used in areas of importance such as promotion panels or carousels where content is left aligned. It should be considered that no more than 1 primary call to action be used per page, to ensure the user understands the hierarchy of the button importance. A pill style button can be utilised for areas of importance where the content is centre aligned or it suits the design. Text should always be Initial caps Din Pro Medium. The appropriate button size will be handled by media queries in build, but for design options are available below.

A secondary call to action should be used when the content the button relates to is of lesser importance than the primary goal of the page. Secondary calls to action should be used for instances such as article cards or product information. As with primary calls to action, secondary are available in 3 sizes and are handled by the CSS.
