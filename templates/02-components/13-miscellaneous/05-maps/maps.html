Maps

Usage and implementation for Maps within Web Design Language.

Introduction

Maps will form the basis of a regional set of results. Maps can be overlayed where space allows with card or tooltip content. In build the relevant regional map service should be used. Maps rely on the Google Maps Javascript API. A suite of JS functions have been created to assist in creating consistent Maps, Markers and InfoWindows. These are displayed in the Javascript code demonstrations. Every map on a page must have a unique ID. Please ensure that every Map, Marker or InfoWindow that is created is stored in a unique variable.

Google Maps API

In order to display a map using the Google Maps Javascript API, and API key must be provided.

Please get in touch with Royal Canin to get access to the official API key.

All Maps, Markers and InfoWindows should be created using the following functions.

These functions must live inside the Google Maps API initMap() function in your scripts file. An example of this can be found in the accompanying maps.jsfile.

Create a Map

  • $selector string - The ID of HTML element where the map should be displayed.
  • $center object - The coordinates for the centre of the Map.
  • $zoom number - The desired zoom level (Defaults to 17).

					var map_royal_canin = create_map( document.getElementById( $selector ), $center, $zoom );

Create a Marker

  • $position object - The coordinates for the position of the Marker.
  • $map object - The Map the Marker should be applied to.

					var marker_royal_canin = create_marker( $position, $map );

Create an InfoWindow

  • $content object - The DOM element that should be used for the InfoWindow contents.
  • $marker object - The Marker the InfoWindow should be applied to.
  • $map object - The Map the Marker is a part of.

					var infobox_first_10 = create_infobox( document.getElementById( 'infobox_first_10' ), marker_first_10, map_first_10 );
