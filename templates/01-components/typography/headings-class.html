Class Headings

Wherever possible, when a Heading tag is used, an accompanying heading class must be applied to set it's size to one of the six predefined levels. These classes should be used in any Component, Feature or Template to set the size of a Heading.

This is due to the HTML5 document outline algorithm allowing multiple content roots on any HTML5 marked up webpage. This means, in short, that multiple <h1> tags can be used if separated by the appropriate HTML5 semantic sectioning elements.

The use of an rc-head--[value] class will set the size of any Heading tag (or indeed any element with that class applied to it) to the corresponding standard Heading size.

As is demonstrated below, the same 6 Heading tags are displayed as on the Default Headings page, but with a different rc-head--[value] class applied in order to change their appearance. In this case, we've effectively reversed the sizes of the Heading levels; <h1> is shown at the size of a <h6>, and so on.
