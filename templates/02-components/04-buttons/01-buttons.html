Buttons

Usage and implementation for Buttons within Web Design Language.

Introduction

Buttons carry out a variety of tasks across projects. Button sizing should be handled with the relevant classes, ensuring best display output across devices. Icons can be used within buttons to highlight a specific action or subject. Please note that icons and buttons are defined separately. However, all icons are sized appropriately to be interchangeable with (or to be used within) buttons.

Examples

A primary call to action should be used in areas of importance, such as promotion panels or carousels where content is left aligned. It should be considered that no more than one primary call to action be used per page, to ensure the user understands the hierarchy of the button importance. A pill style button can be utilised for areas of importance where the content is centre aligned or it suits the design. Text should always be Initial caps Din Pro Medium. The appropriate button size will be handled by media queries in build with design options available below.

A secondary call to action should be used when the content the button relates to is of lesser importance than the primary goal of the page. Secondary calls to action should be used for instances such as article cards or product information. As with primary calls to action, secondary are available in 3 sizes and are handled by the css.
